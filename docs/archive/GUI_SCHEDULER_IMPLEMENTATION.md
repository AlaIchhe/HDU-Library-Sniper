### 2. 实现系统定时任务管理服务 ✓

**文件**: `services/scheduler.py`

**核心类**: `SchedulerService`

**支持平台**:
- ✅ Windows (通过 AutoSchedule.ps1 或 schtasks)
- ✅ Linux (通过 crontab)
- ✅ macOS (通过 crontab)

**主要方法**:
- `configure_task(execute_time, wake_to_run)` - 配置定时任务
- `remove_task()` - 移除定时任务
- `get_task_status()` - 获取任务状态
- `test_execution()` - 测试执行一次

**Windows 实现**:
- 调用 `scripts/AutoSchedule.ps1` 自动注册
- 通过环境变量传递配置（SNIPER_DAILY_AT、SNIPER_WAKE_TO_RUN 等）
- 任务名称：`HDU-Library-Sniper-Daily`
- 账户：`NT AUTHORITY\SYSTEM`

**Linux 实现**:
- 直接操作 crontab
- 自动处理旧任务清理
- 注意：cron 不支持秒级精度，秒数会被忽略

## 🔍 技术亮点

### 1. 跨平台支持

通过 `platform.system()` 自动检测操作系统，调用对应的实现：
- Windows: PowerShell + schtasks
- Linux/macOS: crontab

### 2. 自动路径探测

**Windows**:
- 项目根目录查找 pythonw.exe
- 虚拟环境（venv/.venv）查找
- PATH 环境变量查找
- 避免手动配置路径错误

**Linux**:
- 使用 python3 命令（系统默认）
- 自动设置工作目录

### 3. 错误处理

- 配置失败不关闭对话框，显示错误信息
- 提供详细的错误提示
- 支持重试配置

### 4. 用户友好设计

- 二次确认删除操作
- 测试执行前确认
- 显示进度对话框
- 实时日志输出
- 状态自动刷新

### 5. 环境变量配置

通过环境变量覆盖默认配置：
- `SNIPER_WORKDIR` - 工作目录
- `SNIPER_DAILY_AT` - 执行时间
- `SNIPER_TASK_NAME` - 任务名称
- `SNIPER_WAKE_TO_RUN` - 是否唤醒（Windows）
