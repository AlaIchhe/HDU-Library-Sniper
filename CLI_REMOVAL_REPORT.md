## 🎯 重构目标

**用户只感知 GUI，CLI 完全隐藏在后台作为静默执行引擎**

## ✅ 已完成的变更

### 1. 删除 CLI 交互代码

**删除的文件** (4个，~1200 行):
- ❌ `cli/app.py` - 交互式应用主类
- ❌ `cli/menu.py` - 菜单选择工具
- ❌ `cli/prompts.py` - 输入提示工具（已迁移到 utils/time_utils.py）
- ❌ `cli/views.py` - 格式化输出工具

**保留的文件**:
- ✅ `cli/__init__.py` - 空模块，仅用于向后兼容

### 2. 重构 main.py 入口

**之前** (三种模式):
```python
python main.py              # CLI 交互模式
python main.py --gui        # GUI 模式
python main.py --run-now    # 后台执行
```

**现在** (两种模式):
```python
python main.py              # GUI 模式（默认，用户可见）
python main.py --daemon     # 后台执行（用户不可见）
python main.py --run-now    # 后台执行（兼容旧脚本）
```

**关键变化**:
- ✅ 默认启动 GUI（无需 `--gui` 参数）
- ✅ `--daemon` / `--run-now` 触发后台静默执行
- ✅ 删除 CLI 交互模式入口

### 3. 优化启动脚本

**重命名**:
- `start_gui.bat` → `start.bat`
- `start_gui.sh` → `start.sh`

**新增**:

**启动方式**:

## 🔄 定时任务工作流

### 配置阶段（GUI 界面）

```
用户在 GUI 中点击"配置定时任务"
  ↓
输入执行时间（例如 23:59:55）
  ↓
GUI 自动调用系统 API:
  - Windows: schtasks 创建任务
  - Linux: 写入 crontab
  ↓
配置完成！
```

### 执行阶段（完全隐藏）

```
系统定时任务触发（每天 23:59:55）
  ↓
执行: python main.py --daemon
  ↓
后台守护进程:
  1. 读取 data/plans.yaml
  2. 读取 data/credentials.yaml
  3. 自动续登（如需要）
  4. 执行抢座
  5. 推送通知
  6. 记录日志
  7. 退出（exit code 0/1）
  ↓
用户收到通知（成功/失败）
```

**用户全程无感知后台执行过程！**

## 🔍 技术细节

### 后台执行模式

**触发方式**:
```bash
python main.py --daemon    # 或
python main.py --run-now   # 兼容旧脚本
```

**执行流程**:
1. 读取配置文件（plans.yaml, credentials.yaml）
2. 初始化服务层（AuthService, BookingService）
3. 尝试使用缓存登录，失败则用凭据续登
4. 读取启用的方案
5. 执行抢座（带重试、窗口轮询）
6. 推送通知
7. 记录日志
8. 退出，返回退出码

**退出码**:
- `0` - 至少一个方案成功
- `1` - 全部方案失败
- `2` - 认证失败
- `3` - 没有启用的方案

**特点**:
- ✅ 完全非交互（无任何输入输出）
- ✅ 适合系统定时任务调用
- ✅ 轻量级（不加载 GUI 依赖）
- ✅ 日志文件记录详细信息

### GUI 与后台的交互

```
GUI                     配置文件                  后台守护进程
│                          │                          │
├─ 创建方案 ──────────→  plans.yaml  ◄────────── 读取方案
├─ 保存凭据 ──────────→  credentials.yaml  ◄─── 读取凭据
├─ 配置定时任务 ──────→  任务计划程序/cron  ───→ 定时触发
│                          │                          │
│                          │                    ┌─────▼─────┐
│                          │                    │ 抢座执行   │
│                          │                    │ 推送通知   │
│                          │                    │ 记录日志   │
│                          │                    └───────────┘
│                          │                          │
└─ 查看日志 ◄──────────── logs/  ◄─────────────────┘
```
